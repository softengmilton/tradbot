# instruction.md

## Project Name
AI Trading Chart Analyzer Extension

---

# Project Goal

Build a browser extension that analyzes trading charts directly from the user's screen.

The extension must support:
1. Default Analysis Mode
2. AI Analysis Mode
3. Interactive AI Chat After Analysis

The extension should mainly work with TradingView charts but remain flexible for other chart websites.

The product acts as:
- trading assistant
- chart analyzer
- AI trading co-pilot

NOT as:
- autonomous trading bot
- guaranteed signal provider
- financial advisor

---

# Core Features

## Screenshot-Based Chart Analysis

The extension captures the currently visible browser tab.

Primary API:
- chrome.tabs.captureVisibleTab()

The captured screenshot becomes the primary source for analysis.

---

# Analysis Modes

## 1. Default Mode

Rule-based technical analysis using built-in formulas and indicators.

This mode must:
- work without AI
- be fast
- minimize API cost
- provide deterministic results

### Indicators Used
- EMA
- RSI
- MACD
- ATR
- Volume
- Support/Resistance
- Candlestick patterns

### Default Mode Output Example

Trend: Bullish
Entry Zone: 64,200 - 64,350
Stop Loss: 63,850
Take Profit: 65,200
Confidence: 78%

Reason:
EMA20 crossed above EMA50.
RSI indicates bullish momentum.

---

## 2. AI Mode

Uses AI vision and language models to analyze chart screenshots.

AI Mode capabilities:
- chart understanding
- market structure analysis
- support/resistance interpretation
- breakout detection
- momentum interpretation
- trend analysis
- scenario explanation

### AI Mode Output Example

Market structure remains bullish after reclaiming support.

Momentum appears strong while sellers weaken near resistance.

Potential continuation toward higher resistance zone.

---

# Interactive AI Chat System

## Overview

After the AI generates the initial chart analysis report, the user must be able to continue chatting with the AI about the SAME chart and SAME analysis session.

The AI chat should feel like:
"A professional market analyst continuing a live discussion."

---

# Important Chat Rule

The AI must NEVER forget the current chart context during an active session.

The AI should continue discussing:
- same chart
- same timeframe
- same market structure
- same setup
- same analysis session

until:
- a new screenshot is analyzed
OR
- the session is reset

---

The AI assistant should support:

Examples:

## Trade Discussion

---

## Scenario Analysis

## Educational Guidance

# AI Chat Activation Rules

AI Chat must remain disabled until:
- AI analysis is successfully completed

After report generation:
- chat becomes enabled automatically

---

# AI Chat Workflow

1. User selects AI Mode
2. User clicks Analyze
3. Screenshot captured
4. AI generates report
5. Report displayed
6. AI Chat unlocks
7. User asks follow-up questions
8. AI responds using active chart context

---

# Context Memory Requirements

The backend must store:
- latest screenshot
- latest analysis report
- conversation history
- active symbol
- timeframe
- AI session data

---

# Session Rules

An AI session remains active until:
- user analyzes a new screenshot
- session timeout occurs
- user resets session

---

# New Screenshot Behavior

When a new chart is analyzed:
- old session should be archived
- new session should begin
- new chart becomes primary context

# AI Prompting Rules

The AI should behave like:
- professional trading analyst
- calm assistant
- educational guide

Avoid:
- hype
- emotional language
- fake certainty
- gambling behavior

GOOD:
"Momentum appears bullish while price remains above support."

BAD:
"This trade will definitely pump."

---

# Important Product Rules

## Never Promise Profit

The extension must NEVER:
- guarantee profits
- claim certainty
- claim 100% accuracy
- encourage reckless leverage

The system should instead provide:
- probabilities
- scenarios
- educational insight
- risk awareness

---

# Safety Warnings

Always display warnings such as:

"Trading involves risk."

"AI analysis is not financial advice."

"Use proper risk management."

---

# Performance Requirements

The extension should:
- analyze within 3–10 seconds
- minimize API cost
- avoid unnecessary screenshots
- cache repeated analysis when possible

Chat responses should:
- return within 2–8 seconds
- preserve chart context efficiently

---

# Response Format

All analysis responses should return structured JSON.

Example:

{
  "trend": "Bullish",
  "entry": "64,200 - 64,350",
  "stop_loss": "63,850",
  "take_profit": [
    "65,100",
    "65,800"
  ],
  "confidence": 82,
  "summary": "Bullish continuation likely above support."
}
