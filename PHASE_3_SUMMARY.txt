OVERVIEW
───────────────────────────────────────────────────────────────────

Phase 3 implements an interactive AI chat system that enables users to
ask follow-up questions about their chart analysis while maintaining
the chart context throughout the conversation.

Feature Highlights:
  • Session-based chat with context preservation
  • GPT-4 powered follow-up responses
  • Real-time message send/receive
  • Message history storage per session
  • 60-minute session timeout
  • Professional chat UI with animations
  • Error handling and recovery

TECHNICAL ARCHITECTURE
───────────────────────────────────────────────────────────────────

Message Flow:
  1. User clicks "Enable AI Chat" button
  2. App switches to ChatPanel with sessionId
  3. User types message and presses Enter
  4. ChatPanel sends message via /api/chat
  5. Backend stores user message in session
  6. Backend sends context + history to OpenAI
  7. OpenAI returns response (tokens tracked)
  8. Backend stores AI response in session
  9. ChatPanel displays response in real-time
  10. Conversation continues with full context

Session Context:
  • Session ID: UUID generated on initial analysis
  • Original Analysis: Chart analysis from Phase 2
  • Symbol: Trading pair (e.g., BTCUSD)
  • Message History: Array of {role, content, timestamp}
  • Timeout: 60 minutes of inactivity

───────────────────────────────────────────────────────────────────
DESIGN DECISIONS
───────────────────────────────────────────────────────────────────

1. In-Memory Session Storage
   ✓ Fast access
   ✓ Simple implementation
   ✓ Upgradeable to database later
   ✓ Suitable for MVP with 60-min timeout

2. Full Message History in Session
   ✓ Context always available
   ✓ No additional API calls needed
   ✓ Easy conversation recovery
   ✓ Preparation for export features

3. Separate Chat Model vs Vision Model
   ✓ gpt-4-vision for image analysis
   ✓ gpt-4 for conversation
   ✓ Optimized for each use case
   ✓ More cost-effective

4. Session-Based Instead of User-Based
   ✓ Multiple analyses per user
   ✓ Independent conversations
   ✓ Better scalability
   ✓ Simpler state management
